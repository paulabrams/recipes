# Paul's Recipe Collection

## Format Guide

All recipes in this collection follow a consistent Markdown format. Use `recipe-template.md` (in this folder) as a starting point for new recipes.

### Structure

Every recipe uses these sections in order:

1. **Title** — An H1 heading with the recipe name. An emoji before the name is optional.
2. **Ingredients** — An H2 section. Each ingredient is a checkbox list item (`- [ ]`). When a recipe has distinct ingredient groups (e.g., a rub and a sauce), use H3 subsections to separate them.
3. **Directions** — An H2 section with numbered steps. For longer multi-phase recipes (like smoking or braising), use H3 subsections to break the process into named phases. Steps should be concise but complete.
4. **Tips** — An H2 section with bulleted practical advice: substitutions, timing guidance, technique notes.
5. **Notes** — An H2 section with bulleted metadata: servings, total time, source/attribution, and any other context.

### Conventions

- Ingredients use `- [ ]` checkboxes so they can be ticked off while cooking.
- Temperatures always include °F and are bolded when they appear in directions.
- Internal meat temperatures are called out explicitly.
- Times are specific where possible (e.g., "4–5 hours" rather than "a few hours").
- Directions are numbered; tips and notes are bulleted.
- Keep it practical — write for someone standing at the grill or stove.

### Folder Organization

Recipes are organized into category folders: Appetizers, BBQ, Baking, Brunch, Desserts, Fish, Main Courses, Salads, Sauces and Seasoning, Sides, Smoothies, Juices, Vegetarian, Thai, Macedonian, Takeout, Workout, and Meal Plans for Trips.
